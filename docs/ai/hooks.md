# Recommended Checks And Hooks

This file documents safe checks for future work. Do not install new hook tooling or modify pipelines unless a task explicitly asks for it.

## Before Coding Checks

- `git status --short`
- Read `AGENTS.md`, `docs/ai/context.md`, and `docs/ai/file-map.md`.
- Identify touched surface and narrow verification command.
- Confirm no secrets/env/generated/lock files are needed.

## After Coding Checks

- Re-run `git status --short`.
- Review diff for unrelated changes.
- Run the narrowest relevant tests/builds.
- Update AI docs only if durable context changed.

## Build Checks

- Backend: `dotnet build backend/ThirdWheel.API/ThirdWheel.API.csproj`
- Consumer web: `cd web/triad-web && npm run build`
- Business portal: `cd web/triad-business && npm run build`
- Admin Next.js: `cd admin/nextjs-admin && npm run build`
- iOS: use `./scripts/mobile/run-ios.sh` or the README `xcodebuild` command.

## Test Checks

- Backend unit: `./scripts/run/test-backend.sh unit`
- Backend integration: `./scripts/run/test-backend.sh integration`
- Backend all: `./scripts/run/test-backend.sh all`
- For web/iOS, no existing dedicated automated test suites were found; use build/type/lint/simulator checks unless adding test tooling is approved.

## Lint And Format Checks

- Consumer web: `cd web/triad-web && npm run lint && npm run typecheck`
- Business portal: `cd web/triad-business && npm run lint && npm run typecheck`
- Admin Next.js has build but no explicit lint/typecheck script in `package.json`.
- Do not introduce formatters or rewrite broad files without approval.

## Security Checks

- Search diff for tokens, passwords, connection strings, bearer tokens, precise locations, and private user data.
- Verify auth/role policies when touching `/api/admin/*`, `/api/business/*`, or safety endpoints.
- Confirm CORS and JWT settings remain env-driven.
- Avoid printing `.env*`, local deployment envs, or admin bearer tokens.

## Migration And Seed Checks

- Schema changes should include EF migration files and snapshot updates generated by EF.
- Run backend tests after schema changes.
- Review `seed.ps1` before running; it is destructive to seeded data.
- Do not change seed defaults or migrations as collateral.

## API Contract Checks

- If backend DTOs/routes change, update web types/API services and iOS models/API calls as needed.
- Check `README.md` endpoint docs and `docs/ai/context.md` for durable contract changes.
- Prefer additive API changes unless explicitly asked for breaking changes.

## UI Validation Checks

- Run lint/typecheck/build in the touched web app.
- Check loading, empty, error, and authenticated/unauthenticated states.
- Avoid changing visual systems across consumer, business, and admin surfaces as collateral.
- After significant frontend changes, test in a browser if available.

## Mobile Validation Checks

- Run the iOS simulator helper when practical.
- Confirm `APIBaseURL` behavior remains simulator/local-device compatible.
- Verify Keychain/session and location permission behavior if touched.
- Avoid Xcode project edits unless adding/removing files requires it.

## PR Validation Checks

- Relevant tests/builds/lints pass or are clearly reported as not run.
- Diff excludes secrets, generated files, env files, lock files, unrelated refactors, and deployment churn.
- API/client contracts are aligned.
- Risk and follow-up work are stated.
